Router-on-a-Stick (ROAS) Lab

Topology

This lab simulates a small office network with three departments, each on a separate VLAN, connected using Router-on-a-Stick (ROAS) for inter-VLAN routing. ROAS uses a single router interface with subinterfaces to route traffic between VLANs via a trunk link. The topology includes:

  • 1 Router (Router0)
  • 2 Switches (Switch0, Switch1)
  • 3 VLANs:
    • VLAN 10 (IT Dept): 4 PCs (PC0, PC1 on Switch0; PC6, PC7 on Switch1)
    • VLAN 20 (Sales): 2 PCs (PC2, PC3 on Switch0)
    • VLAN 30 (Marketing): 2 PCs (PC4, PC6 on Switch1)
  • Trunk links:
    • Switch0 Fa0/6 to Router0 G0/0
    • Switch0 Fa0/5 to Switch1 Fa0/5

Introduction

This lab demonstrates Router-on-a-Stick (ROAS) for inter-VLAN routing, a technique that uses a single router interface with subinterfaces to route traffic between VLANs. ROAS is cost-effective for small networks as it requires only one physical router interface.

The objective was to configure VLANs on the switches, set up subinterfaces on the router for inter-VLAN routing via a trunk link, and verify connectivity between devices in different VLANs using ping tests.

Configurations

IP Address Assignments

The network 192.168.5.0/24 was subnetted as follows:

  • VLAN 10 (IT Dept): 192.168.5.0/26 (192.168.5.1 - 192.168.5.62)
  • VLAN 20 (Sales): 192.168.5.64/26 (192.168.5.65 - 192.168.5.126)
  • VLAN 30 (Marketing): 192.168.5.128/26 (192.168.5.129 - 192.168.5.190)

Note on IP Assignment Logic: For each VLAN, the first usable IP address was assigned to the PCs to ensure a consistent addressing scheme. The last usable IP address was assigned to the router’s subinterface, following the convention of reserving the highest address for the gateway.

Device IPs:

  • VLAN 10 (IT Dept):
    • PC0 (Switch0): 192.168.5.1, Default Gateway: 192.168.5.62
    • PC1 (Switch0): 192.168.5.2, Default Gateway: 192.168.5.62
    • PC6 (Switch1): 192.168.5.5, Default Gateway: 192.168.5.62
    • PC7 (Switch1): 192.168.5.6, Default Gateway: 192.168.5.62
    • Router0 G0/0.10: 192.168.5.62
  • VLAN 20 (Sales):
    • PC2 (Switch0): 192.168.5.65, Default Gateway: 192.168.5.126
    • PC3 (Switch0): 192.168.5.66, Default Gateway: 192.168.5.126
    • Router0 G0/0.20: 192.168.5.126
  • VLAN 30 (Marketing):
    • PC4 (Switch1): 192.168.5.129, Default Gateway: 192.168.5.190
    • PC6 (Switch1): 192.168.5.130, Default Gateway: 192.168.5.190
    • Router0 G0/0.30: 192.168.5.190

Router Configurations

Router0 (ROAS):

Router0> enable
Router0# configure terminal
Router0(config)# interface GigabitEthernet0/0.10
Router0(config-subif)# encapsulation dot1Q 10
Router0(config-subif)# ip address 192.168.5.62 255.255.255.192
Router0(config-subif)# no shutdown
Router0(config-subif)# exit
Router0(config)# interface GigabitEthernet0/0.20
Router0(config-subif)# encapsulation dot1Q 20
Router0(config-subif)# ip address 192.168.5.126 255.255.255.192
Router0(config-subif)# no shutdown
Router0(config-subif)# exit
Router0(config)# interface GigabitEthernet0/0.30
Router0(config-subif)# encapsulation dot1Q 30
Router0(config-subif)# ip address 192.168.5.190 255.255.255.192
Router0(config-subif)# no shutdown
Router0(config-subif)# exit
Router0(config)# interface GigabitEthernet0/0
Router0(config-if)# no shutdown
Router0(config-if)# exit

PC Configurations

Each PC was configured with the first usable IP in its VLAN and the router’s subinterface as the default gateway:

  • PC0 (Switch0, VLAN 10): IP 192.168.5.1, Subnet Mask 255.255.255.192, Gateway 192.168.5.62
  • PC1 (Switch0, VLAN 10): IP 192.168.5.2, Subnet Mask 255.255.255.192, Gateway 192.168.5.62
  • PC2 (Switch0, VLAN 20): IP 192.168.5.65, Subnet Mask 255.255.255.192, Gateway 192.168.5.126
  • PC3 (Switch0, VLAN 20): IP 192.168.5.66, Subnet Mask 255.255.255.192, Gateway 192.168.5.126
  • PC4 (Switch1, VLAN 30): IP 192.168.5.129, Subnet Mask 255.255.255.192, Gateway 192.168.5.190
  • PC6 (Switch1, VLAN 30): IP 192.168.5.5, Subnet Mask 255.255.255.192, Gateway 192.168.5.190
  • PC7 (Switch1, VLAN 10): IP 192.168.5.6, Subnet Mask 255.255.255.192, Gateway 192.168.5.62

Results

The lab was successfully completed:

  • Devices in different VLANs could communicate via ROAS using a single trunk link between Switch0 and Router0.
  • Ping from PC0 (VLAN 10, 192.168.5.1 on Switch0) to PC4 (VLAN 30, 192.168.5.129 on Switch1) succeeded with 100% packet delivery, as shown below.
  • VLAN segmentation isolated broadcast traffic, improving network performance.
  • 64 addresses (192.168.5.192 - 192.168.5.255) remain for future expansion.
