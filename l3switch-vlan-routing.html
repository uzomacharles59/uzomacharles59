Layer 3 Switch VLAN Routing Lab

Topology

This lab simulates a small office network with three departments, each on a separate VLAN, connected using a Layer 3 switch for inter-VLAN routing. The Layer 3 switch uses Switch Virtual Interfaces (SVIs) to route traffic between VLANs, providing a more scalable solution than Router-on-a-Stick (ROAS). The topology includes:

  • 1 Layer 3 Switch (Multilayer Switch0)
  • 1 Layer 2 Switch (Switch0)
  • 3 VLANs:
    • VLAN 10 (IT Dept): 4 PCs (PC0, PC1 on Switch0; PC6, PC7 on Multilayer Switch0)
    • VLAN 20 (Sales): 2 PCs (PC2, PC3 on Switch0)
    • VLAN 30 (Marketing): 2 PCs (PC4, PC5 on Multilayer Switch0)
  • Trunk link: Switch0 Fa0/5 to Multilayer Switch0 G1/0/1

Introduction

This lab demonstrates inter-VLAN routing using a Layer 3 switch, a more scalable and efficient method compared to Router-on-a-Stick (ROAS). A Layer 3 switch performs routing using Switch Virtual Interfaces (SVIs), eliminating the need for a separate router and reducing bottlenecks.

The objective was to configure VLANs on the switches, set up SVIs on the Layer 3 switch for inter-VLAN routing, and verify connectivity between devices in different VLANs using ping tests.

Configurations

IP Address Assignments

The network 192.168.5.0/24 was subnetted as follows:

  • VLAN 10 (IT Dept): 192.168.5.0/26 (192.168.5.1 - 192.168.5.62)
  • VLAN 20 (Sales): 192.168.5.64/26 (192.168.5.65 - 192.168.5.126)
  • VLAN 30 (Marketing): 192.168.5.128/26 (192.168.5.129 - 192.168.5.190)

Note on IP Assignment Logic: For each VLAN, the first usable IP address was assigned to the PCs to ensure a consistent addressing scheme. The last usable IP address was assigned to the Layer 3 switch’s SVI, following the convention of reserving the highest address for the gateway.

Device IPs:

  • VLAN 10 (IT Dept):
    • PC0 (Switch0): 192.168.5.1, Default Gateway: 192.168.5.62
    • PC1 (Switch0): 192.168.5.2, Default Gateway: 192.168.5.62
    • PC6 (Multilayer Switch0): 192.168.5.5, Default Gateway: 192.168.5.62
    • PC7 (Multilayer Switch0): 192.168.5.6, Default Gateway: 192.168.5.62
    • Multilayer Switch0 VLAN 10 SVI: 192.168.5.62
  • VLAN 20 (Sales):
    • PC2 (Switch0): 192.168.5.65, Default Gateway: 192.168.5.126
    • PC3 (Switch0): 192.168.5.66, Default Gateway: 192.168.5.126
    • Multilayer Switch0 VLAN 20 SVI: 192.168.5.126
  • VLAN 30 (Marketing):
    • PC4 (Multilayer Switch0): 192.168.5.129, Default Gateway: 192.168.5.190
    • PC5 (Multilayer Switch0): 192.168.5.130, Default Gateway: 192.168.5.190
    • Multilayer Switch0 VLAN 30 SVI: 192.168.5.190

Layer 3 Switch Configurations

Multilayer Switch0 (Layer 3):

  MultilayerSwitch0> enable
  MultilayerSwitch0# configure terminal
  MultilayerSwitch0(config)# vlan 10
  MultilayerSwitch0(config-vlan)# name IT_Dept
  MultilayerSwitch0(config-vlan)# exit
  MultilayerSwitch0(config)# vlan 20
  MultilayerSwitch0(config-vlan)# name Sales
  MultilayerSwitch0(config-vlan)# exit
  MultilayerSwitch0(config)# vlan 30
  MultilayerSwitch0(config-vlan)# name Marketing
  MultilayerSwitch0(config-vlan)# exit
  MultilayerSwitch0(config)# interface range GigabitEthernet1/0/4 - 5
  MultilayerSwitch0(config-if-range)# switchport mode access
  MultilayerSwitch0(config-if-range)# switchport access vlan 10
  MultilayerSwitch0(config-if-range)# exit
  MultilayerSwitch0(config)# interface range GigabitEthernet1/0/2 - 3
  MultilayerSwitch0(config-if-range)# switchport mode access
  MultilayerSwitch0(config-if-range)# switchport access vlan 30
  MultilayerSwitch0(config-if-range)# exit
  MultilayerSwitch0(config)# interface GigabitEthernet1/0/1
  MultilayerSwitch0(config-if)# switchport mode trunk
  MultilayerSwitch0(config-if)# switchport trunk allowed vlan 10,20,30
  MultilayerSwitch0(config-if)# exit
  MultilayerSwitch0(config)# ip routing
  MultilayerSwitch0(config)# interface vlan 10
  MultilayerSwitch0(config-if)# ip address 192.168.5.62 255.255.255.192
  MultilayerSwitch0(config-if)# no shutdown
  MultilayerSwitch0(config-if)# exit
  MultilayerSwitch0(config)# interface vlan 20
  MultilayerSwitch0(config-if)# ip address 192.168.5.126 255.255.255.192
  MultilayerSwitch0(config-if)# no shutdown
  MultilayerSwitch0(config-if)# exit
  MultilayerSwitch0(config)# interface vlan 30
  MultilayerSwitch0(config-if)# ip address 192.168.5.190 255.255.255.192
  MultilayerSwitch0(config-if)# no shutdown
  MultilayerSwitch0(config-if)# exit

Note on `no switchport`: The `no switchport` command was not used in this lab because inter-VLAN routing is handled using Switch Virtual Interfaces (SVIs) on Multilayer Switch0. SVIs (`interface vlan`) allow the switch to route between VLANs without converting physical interfaces to routed ports. Physical interfaces remain in Layer 2 mode (access or trunk) to handle VLAN traffic. The `no switchport` command would be used if we had a direct Layer 3 link to another device (e.g., a router for external connectivity), converting a physical interface into a routed port with its own IP address.

Switch0 (Layer 2):

  Switch0# configure terminal
  Switch0(config)# vlan 10

  Switch0(config-if)# switchport mode trunk
  Switch0(config-if)# switchport trunk allowed vlan 10,20,30

PC Configurations

Each PC was configured with the first usable IP in its VLAN and the Layer 3 switch’s SVI as the default gateway:

  • PC0 (Switch0, VLAN 10): IP 192.168.5.1, Subnet Mask 255.255.255.192, Gateway 192.168.5.62
  • PC1 (Switch0, VLAN 10): IP 192.168.5.2, Subnet Mask 255.255.255.192, Gateway 192.168.5.62
  • PC2 (Switch0, VLAN 20): IP 192.168.5.65, Subnet Mask 255.255.255.192, Gateway 192.168.5.126
  • PC3 (Switch0, VLAN 20): IP 192.168.5.66, Subnet Mask 255.255.255.192, Gateway 192.168.5.126
  • PC4 (Multilayer Switch0, VLAN 30): IP 192.168.5.129, Subnet Mask 255.255.255.192, Gateway 192.168.5.190
  • PC5 (Multilayer Switch0, VLAN 30): IP 192.168.5.130, Subnet Mask 255.255.255.192, Gateway 192.168.5.190
  • PC6 (Multilayer Switch0, VLAN 10): IP 192.168.5.5, Subnet Mask 255.255.255.192, Gateway 192.168.5.62
  • PC7 (Multilayer Switch0, VLAN 10): IP 192.168.5.6, Subnet Mask 255.255.255.192, Gateway 192.168.5.62

Results

The lab was successfully completed:

  • Devices in different VLANs could communicate via the Layer 3 switch using SVIs for routing.
  • Ping from PC0 (VLAN 10, 192.168.5.1 on Switch0) to PC4 (VLAN 30, 192.168.5.129 on Multilayer Switch0) succeeded with 100% packet delivery, as shown below.
  • VLAN segmentation isolated broadcast traffic, improving network performance.
  • 64 addresses (192.168.5.192 - 192.168.5.255) remain for future expansion.
