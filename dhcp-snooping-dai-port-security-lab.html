DHCP Snooping, DAI, and Port Security Lab

Overview

This lab demonstrates Layer 2 security features—DHCP Snooping, Dynamic ARP Inspection (DAI), and Port Security—using a Cisco 3560 Layer-3 switch and two Cisco 2960 access switches. The setup includes a legitimate DHCP server in VLAN 30 and four PCs across VLANs 10 and 20, with a management VLAN 99.

Objective: Prevent rogue DHCP servers, ARP spoofing, and unauthorized access by configuring DHCP Snooping to filter DHCP traffic, DAI to validate ARP packets, and Port Security to restrict MAC addresses on access ports.

The lab showcases a secure internal network with inter-VLAN routing handled by the Core L3 switch, ensuring only trusted devices can operate.

Topology

  • Devices:
    • Core (3560) - Cisco 3560 Layer-3 switch.
    • ASW1, ASW2 - Cisco 2960 Access switches.
    • DHCP Server - PT "Server" device.
    • PC0, PC1, PC2, PC3 - End hosts.
  • Connections:
    • Core Gi0/1 ↔ ASW1 Gi0/1 (trunk, all VLANs).
    • Core Gi0/2 ↔ ASW2 Gi0/2 (trunk, all VLANs).
    • ASW1 F0/1 ↔ ASW2 F0/1 (trunk, all VLANs).
    • DHCP Server Fa0 ↔ Core Fa0/1 (access VLAN 30).
    • PC0 ↔ ASW1 Fa0/2 (access VLAN 10).
    • PC1 ↔ ASW1 Fa0/3 (access VLAN 20).
    • PC2 ↔ ASW2 Fa0/2 (access VLAN 10).
    • PC3 ↔ ASW2 Fa0/3 (access VLAN 20).
  • IP Addressing:
    • VLAN 10 (Users1): 10.10.10.0/24, GW: 10.10.10.1 (Core SVI).
    • VLAN 20 (Users2): 10.10.20.0/24, GW: 10.10.20.1 (Core SVI).
    • VLAN 30 (Servers): 10.10.30.0/24, GW: 10.10.30.1 (Core SVI), DHCP Server: 10.10.30.10.
    • VLAN 99 (Management): 10.10.99.0/24, GW: 10.10.99.1 (Core SVI), ASW1: 10.10.99.2, ASW2: 10.10.99.3.

PCs

PCs configured to obtain IP addresses automatically via DHCP.

Verification

Verify DHCP Snooping bindings with show ip dhcp snooping binding on ASW1 and ASW2 to ensure only trusted DHCP assignments are recorded after PCs request IPs. Check DAI statistics with show ip arp inspection statistics to confirm ARP packet validation after testing with a spoofed ARP packet. Verify Port Security status with show port-security to confirm MAC address restrictions after triggering a violation.

Connectivity Test

Test connectivity by pinging between PCs in VLAN 10 (PC0, PC2) and VLAN 20 (PC1, PC3) via the Core L3 switch. Verify all PCs receive IPs from the DHCP server (e.g., PC0: 10.10.10.100+, PC1: 10.10.20.100+). Trigger a Port Security violation by connecting a switch to ASW1 Fa0/2 with PC0 and at least one additional device, then revert by removing the switch and issuing shutdown and no shutdown on Fa0/2 (note: if the port goes down again, clear sticky MACs with clear port-security sticky and retest).
