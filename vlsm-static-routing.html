Enterprise Network with VLSM and Static Routing

Topology

This project involves a multi-LAN network connecting two sites with a point-to-point router link. The topology includes:

  • 2 Routers (Router0, Router1)
  • 4 Switches (Switch0, Switch1, Switch2, Switch3)
  • 4 LANs: LAN 1 (60 hosts), LAN 2 (47 hosts), LAN 3 (15 hosts), LAN 4 (8 hosts)
  • Point-to-point serial link between Router0 and Router1

About VLSM and Static Routing

Variable Length Subnet Masking (VLSM) was used to efficiently allocate IP addresses from the 192.168.5.0/24 network. VLSM allows subnets of different sizes, minimizing IP waste by tailoring each subnet to the exact number of hosts needed.

Static Routing was implemented to enable communication between LANs. Static routes were configured on Router0 and Router1, directing traffic between the two sites via the point-to-point link. This approach ensures predictable routing in a small network while demonstrating foundational routing skills.

Configurations

IP Address Assignments

Using VLSM, the network was subnetted as follows:

  • LAN 1 (60 hosts): 192.168.5.0/26 (192.168.5.1 - 192.168.5.62)
  • LAN 2 (47 hosts): 192.168.5.64/26 (192.168.5.65 - 192.168.5.126)
  • LAN 3 (15 hosts): 192.168.5.128/27 (192.168.5.129 - 192.168.5.158)
  • LAN 4 (8 hosts): 192.168.5.160/28 (192.168.5.161 - 192.168.5.174)
  • Router Link: 192.168.5.176/30 (192.168.5.177 - 192.168.5.178)

Note on IP Assignment Logic: For each LAN, the first usable IP address was assigned to the PC to ensure a consistent and predictable addressing scheme for end devices. The last usable IP address was assigned to the router interface, as it’s a common convention to reserve the highest address for the gateway, making it easily identifiable in network documentation and troubleshooting.

Device IPs:

  • PC0 (LAN 1): 192.168.5.1, Default Gateway: 192.168.5.62 (Router0 Gig0/0)
  • PC1 (LAN 2): 192.168.5.65, Default Gateway: 192.168.5.126 (Router0 Gig0/1)
  • PC2 (LAN 3): 192.168.5.129, Default Gateway: 192.168.5.158 (Router1 Gig0/0)
  • PC3 (LAN 4): 192.168.5.161, Default Gateway: 192.168.5.174 (Router1 Gig0/1)
  • Router0 g0/2: 192.168.5.177
  • Router1 g0/2: 192.168.5.178

Router Configurations

Router0:

Router0> enable
Router0# configure terminal
Router0(config)# interface GigabitEthernet0/0
Router0(config-if)# ip address 192.168.5.62 255.255.255.192
Router0(config-if)# no shutdown
Router0(config-if)# exit
Router0(config)# interface GigabitEthernet0/1
Router0(config-if)# ip address 192.168.5.126 255.255.255.192
Router0(config-if)# no shutdown
Router0(config-if)# exit
Router0(config)# interface g0/2
Router0(config-if)# ip address 192.168.5.177 255.255.255.252

PC Configurations

Each PC was configured with the first usable IP in its subnet and the router interface as the default gateway:

  • PC0: IP 192.168.5.1, Subnet Mask 255.255.255.192, Gateway 192.168.5.62
  • PC1: IP 192.168.5.65, Subnet Mask 255.255.255.192, Gateway 192.168.5.126
  • PC2: IP 192.168.5.129, Subnet Mask 255.255.255.224, Gateway 192.168.5.158
  • PC3: IP 192.168.5.161, Subnet Mask 255.255.255.240, Gateway 192.168.5.174

Results

The network was successfully configured and tested:

  • All PCs could ping each other, confirming static routing worked as expected.
  • Example: Ping from PC0 (192.168.5.1) to PC2 (192.168.5.129) succeeded with 100% packet delivery, as shown in the image below.
  • VLSM ensured efficient IP allocation, leaving 76 addresses (192.168.5.180 - 192.168.5.255) for future expansion.
  • The design demonstrated scalability and reliability for a small enterprise network.
