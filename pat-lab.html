Port Address Translation (PAT) Lab

Overview

This lab demonstrates Port Address Translation (PAT), a form of Network Address Translation that maps multiple private IP addresses to a single public IP using unique port numbers, resolving pool exhaustion issues seen in the Dynamic NAT Lab. The topology extends the Dynamic NAT setup by adding a third PC, which would fail to connect due to limited pool IPs.

What is PAT? PAT, also known as NAT overload, allows multiple internal devices to share a single public IP by assigning unique port numbers to each session. This lab configures Router R1 to use PAT, enabling PC0, PC1, and PC2 to access Server1 using one public IP (203.0.113.10).

The lab addresses the limitation of Dynamic NAT’s pool size, ensuring all three PCs can communicate with the external server simultaneously.

Topology

  • Devices:
    • Router R1 - Cisco 2911 (PAT device).
    • Switch S1 - Cisco 2950.
    • PC0, PC1, PC2 - Internal devices.
    • Server1 - External device.
  • Connections:
    • PC0 (F0) to Switch S1 F0/1.
    • PC1 (F0) to Switch S1 F0/2.
    • PC2 (F0) to Switch S1 F0/3.
    • Switch S1 G0/1 to Router R1 G0/1 (Inside interface).
    • Router R1 G0/0 (Outside interface) to Server1.
  • IP Addressing:
    • Inside network: 192.168.1.0/24
      • PC0: 192.168.1.10, Gateway: 192.168.1.1.
      • PC1: 192.168.1.11, Gateway: 192.168.1.1.
      • PC2: 192.168.1.12, Gateway: 192.168.1.1.
      • Router R1 G0/1: 192.168.1.1.
    • Outside network: 203.0.113.0/24
      • Router R1 G0/0: 203.0.113.1.
      • Server1: 203.0.113.2, Gateway: 203.0.113.1.
    • PAT Configuration:
      • Single public IP: 203.0.113.10 (with overload).

Configurations

PC0

interface Fa0
 ip address 192.168.1.10 255.255.255.0
 default-gateway 192.168.1.1
exit

PC1

interface Fa0
 ip address 192.168.1.11 255.255.255.0
 default-gateway 192.168.1.1
exit

PC2

interface Fa0
 ip address 192.168.1.12 255.255.255.0
 default-gateway 192.168.1.1
exit

Switch S1

interface range F0/1 - 3
 switchport mode access
 no shutdown
exit
interface G0/1
 switchport mode access
 no shutdown
exit

Router R1

interface G0/1
 ip address 192.168.1.1 255.255.255.0
 no shutdown
exit
interface G0/0
 ip address 203.0.113.1 255.255.255.0
 no shutdown
exit
access-list 1 permit 192.168.1.0 0.0.0.255
ip nat pool PAT_POOL 203.0.113.10 203.0.113.10 netmask 255.255.255.0
ip nat inside source list 1 pool PAT_POOL overload
interface G0/1
 ip nat inside
exit
interface G0/0
 ip nat outside
exit

Server1

interface Fa0
 ip address 203.0.113.2 255.255.255.0
 default-gateway 203.0.113.1
exit

Verification

Verify the PAT configuration on Router R1 using show ip nat translations to confirm multiple private IPs (192.168.1.10, 192.168.1.11, 192.168.1.12) mapped to 203.0.113.10 with unique ports. Check NAT statistics with show ip nat statistics to review pool usage and translation activity.

Connectivity Test

Successful pings from PC0, PC1, and PC2 to Server1 (203.0.113.2) verify outbound connectivity, with PAT mapping all to 203.0.113.10 using unique ports.
