OSPF Lab

Overview

This lab demonstrates the configuration and operation of Open Shortest Path First (OSPF), a link-state routing protocol that uses the Dijkstra algorithm to calculate the shortest path tree. OSPF supports hierarchical design, fast convergence, and dynamic routing updates. The lab features four routers with GigabitEthernet and FastEthernet links, connected to two LANs via switches, and an external ISP router for internet access. The focus is on OSPF’s ability to prioritize faster links, propagate a default route, and maintain neighbor relationships.

What is OSPF? OSPF advertises link-state information to build a complete topology database, calculates the shortest path based on cost (inversely proportional to bandwidth), and supports areas for scalability. Key features include support for VLSM, route summarization, and the ability to handle complex networks with multiple paths.

In this lab, the OSPF reference bandwidth was adjusted to 10,000 Mbps using the auto-cost reference-bandwidth 10000 command, ensuring that GigabitEthernet links (cost 10) are preferred over FastEthernet links (cost 100) for routing decisions, aligning with the mixed link speeds in the topology.

The lab verifies OSPF’s dynamic routing, shortest path selection (favoring GigabitEthernet), and default route propagation from Router0 to other routers.

OSPF Verification

The OSPF neighbor table on Router0 shows active neighbors (Router1 and Router2), confirming successful adjacency. The routing table displays OSPF-learned routes (marked with "O"), including all LANs, inter-router links, loopbacks, and the default route propagated from Router0. With the adjusted reference bandwidth, Router3 prefers the path to Router0 via Router1 (GigabitEthernet link, cost 10) over Router2 (FastEthernet link, cost 100).

Note on Cost Calculation: OSPF calculates the cost as the cumulative path cost from the local router to the destination, using the formula Cost = Reference Bandwidth (10,000 Mbps) / Interface Bandwidth (in Mbps). For example, GigabitEthernet links have a cost of 10, and FastEthernet links have a cost of 100. The total cost includes the local interface cost plus the cost advertised by the next-hop router, with an additional intra-area base cost (typically 10) added for routes within the same area. This results in observed costs like 110 (e.g., 10 + 100) and 200 (e.g., 100 + 100), reflecting the shortest paths through the mixed-speed topology.

Connectivity Test

A successful ping from PC0 (192.168.5.2) to PC1 (192.168.6.2) and Router3 Loopback0 (10.0.3.1) verifies that OSPF is routing traffic correctly, with the shortest path via the GigabitEthernet link between Router0 and Router1.
