EtherChannel Lab

Overview

This lab focuses on configuring multiple EtherChannel links in a complex Layer 2 topology to enhance bandwidth and redundancy. EtherChannel, also known as link aggregation, is a critical CCNA topic that bundles physical links into a single logical link. In this lab, I set up two EtherChannels using LACP between a core switch and two distribution switches, integrated VLANs, and implemented inter-VLAN routing with a Router-on-a-Stick configuration. A redundant link between the distribution switches introduces STP interaction, adding complexity.

What is EtherChannel? EtherChannel is a technology that allows grouping of several physical Ethernet links into a single logical link, providing higher throughput and fault tolerance. It uses protocols like LACP (Link Aggregation Control Protocol) or PAgP (Port Aggregation Protocol) to negotiate the bundling. Benefits include load balancing across links, increased bandwidth, and automatic failover if one link fails.

The lab demonstrates advanced configuration, verification, and testing of EtherChannel in a multi-switch environment, ensuring connectivity across VLANs through inter-VLAN routing, and showcasing the interplay with STP for loop prevention.

Topology

  • Devices:
    • Switch0 (Cisco 2950, core), Switch1 (Cisco 2950, distribution), Switch2 (Cisco 2950, distribution).
    • Router0 (Cisco 2911) for inter-VLAN routing.
    • Switch1: PC0 (VLAN 10), PC1 (VLAN 20), PC2 (VLAN 30).
    • Switch2: PC3 (VLAN 10), PC4 (VLAN 20), PC5 (VLAN 30).
  • Connections:
    • EtherChannel 1 (Port-channel1): Switch0 F0/1-F0/2 to Switch1 F0/1-F0/2 (trunk).
    • EtherChannel 2 (Port-channel2): Switch0 F0/3-F0/4 to Switch2 F0/1-F0/2 (trunk).
    • Redundant Link: Switch1 F0/3 to Switch2 F0/3 (trunk, blocked by STP).
    • Switch0 F0/5 to Router0 G0/0/0 (trunk).
    • Switch1 F0/4 to PC0, F0/5 to PC1, F0/6 to PC2.
    • Switch2 F0/4 to PC3, F0/5 to PC4, F0/6 to PC5.
  • VLANs: VLAN 10 (IT Dept), VLAN 20 (Sales), VLAN 30 (HR).
  • IP Addressing:
    • VLAN 10: 192.168.10.0/26, Gateway 192.168.10.62.
    • VLAN 20: 192.168.10.64/26, Gateway 192.168.10.126.
    • VLAN 30: 192.168.10.128/26, Gateway 192.168.10.190.

EtherChannel Verification

The EtherChannels were verified using the `show etherchannel summary` command. Port-channel1 (Switch0-Switch1) and Port-channel2 (Switch0-Switch2) are active, with both links in each bundle operational using LACP.

STP Analysis

The redundant link between Switch1 F0/3 and Switch2 F0/3 is blocked by STP to prevent a loop in the topology. Switch0 acts as the root bridge by default, with Port-channel1 and Port-channel2 as root ports.

Inter-VLAN Routing Test

A successful ping from PC0 (VLAN 10, 192.168.10.2) on Switch1 to PC5 (VLAN 30, 192.168.10.131) on Switch2 confirms that inter-VLAN routing is working, with traffic passing through the EtherChannel links.
