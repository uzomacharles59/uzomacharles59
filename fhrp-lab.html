FHRP Lab

Overview

This lab demonstrates the configuration and operation of First Hop Redundancy Protocol (FHRP) using Hot Standby Router Protocol (HSRP), which provides gateway redundancy by allowing multiple routers to share a virtual IP address. The setup features two routers (Router0 and Router1) providing redundant gateways for two VLANs, with OSPF for dynamic routing and an external router (Router2) simulating an ISP. Two switches connect the VLANs, and four PCs test connectivity and failover.

What is HSRP? HSRP allows multiple routers to act as a single virtual router, ensuring high availability for hosts. One router is active, handling traffic, while the other is on standby, ready to take over if the active router fails.

The lab uses HSRP to assign virtual IPs as default gateways (192.168.10.1 for VLAN 10, 192.168.20.1 for VLAN 20), with Router0 active for VLAN 10 and Router1 active for VLAN 20. OSPF is configured with a reference bandwidth of 10,000 Mbps to prioritize faster links, and Router2 advertises a default route for external connectivity.

Key objectives include verifying HSRP failover, inter-VLAN routing, and external connectivity to Router2’s network (203.0.113.0/24).

Topology

  • Devices:
    • Router0, Router1, Router2 - Cisco 2911.
    • Switch0, Switch1 - Cisco 2950.
    • PC0, PC1 (connected to Switch0), PC2, PC3 (connected to Switch1).
  • Connections:
    • Router0 G0/0 to Switch0 F0/1 (VLAN 10).
    • Router0 G0/1 to Switch1 F0/1 (VLAN 20).
    • Router1 G0/0 to Switch0 F0/2 (VLAN 10).
    • Router1 G0/1 to Switch1 F0/2 (VLAN 20).
    • Router0 G0/2 to Router1 G0/2 (inter-router link).
    • Router0 S0/0/0 to Router2 S0/0/0 (external link).
    • Router1 S0/0/0 to Router2 S0/0/1 (external link).
    • Switch0 F0/3 to PC0, F0/4 to PC1 (VLAN 10).
    • Switch1 F0/3 to PC2, F0/4 to PC3 (VLAN 20).
    • Switch0 F0/5 to Switch1 F0/5 (trunk link).
  • IP Addressing:
    • VLAN 10: 192.168.10.0/24 (Virtual IP: 192.168.10.1, Router0 G0/0: 192.168.10.2, Router1 G0/0: 192.168.10.3).
    • VLAN 20: 192.168.20.0/24 (Virtual IP: 192.168.20.1, Router0 G0/1: 192.168.20.2, Router1 G0/1: 192.168.20.3).
    • Router0 G0/2 to Router1 G0/2: 192.168.100.0/30 (Router0: 192.168.100.1, Router1: 192.168.100.2).
    • Router0 S0/0/0 to Router2 S0/0/0: 192.168.0.0/30 (Router0: 192.168.0.1, Router2: 192.168.0.2).
    • Router1 S0/0/0 to Router2 S0/0/1: 192.168.0.4/30 (Router1: 192.168.0.5, Router2: 192.168.0.6).
    • Router2 Loopback0: 203.0.113.0/24 (Router2: 203.0.113.1).
    • PC0: 192.168.10.10, PC1: 192.168.10.11 (Gateway: 192.168.10.1).
    • PC2: 192.168.20.10, PC3: 192.168.20.11 (Gateway: 192.168.20.1).

Configurations

Router0

      interface G0/0
       ip address 192.168.10.2 255.255.255.0
       standby 1 ip 192.168.10.1
       standby 1 priority 110
       standby 1 preempt
       no shutdown
      exit
      interface G0/1
       ip address 192.168.20.2 255.255.255.0
       standby 2 ip 192.168.20.1
       standby 2 priority 90
       no shutdown
      exit
      interface G0/2
       ip address 192.168.100.1 255.255.255.252
       no shutdown
      exit
      interface S0/0/0
       ip address 192.168.0.1 255.255.255.252
       no shutdown
      exit
      router ospf 1
       auto-cost reference-bandwidth 10000
       passive-interface G0/0
       passive-interface G0/1
       network 192.168.10.0 0.0.0.255 area 0
       network 192.168.20.0 0.0.0.255 area 0
       network 192.168.100.0 0.0.0.3 area 0
       network 192.168.0.0 0.0.0.3 area 0
      exit

Router1

      interface G0/0
       ip address 192.168.10.3 255.255.255.0
       standby 1 ip 192.168.10.1
       standby 1 priority 90
       no shutdown
      exit
      interface G0/1
       ip address 192.168.20.3 255.255.255.0
       standby 2 ip 192.168.20.1
       standby 2 priority 110
       standby 2 preempt
       no shutdown
      exit
      interface G0/2
       ip address 192.168.100.2 255.255.255.252
       no shutdown
      exit
      interface S0/0/0
       ip address 192.168.0.5 255.255.255.252
       no shutdown
      exit
      router ospf 1
       auto-cost reference-bandwidth 10000
       passive-interface G0/0
       passive-interface G0/1
       network 192.168.10.0 0.0.0.255 area 0
       network 192.168.20.0 0.0.0.255 area 0
       network 192.168.100.0 0.0.0.3 area 0
       network 192.168.0.0 0.0.0.3 area 0
      exit

Router2

      interface S0/0/0
       ip address 192.168.0.2 255.255.255.252
       no shutdown
      exit
      interface S0/0/1
       ip address 192.168.0.6 255.255.255.252
       no shutdown
      exit
      interface Loopback0
       ip address 203.0.113.1 255.255.255.0
       no shutdown
      exit
      router ospf 1
       auto-cost reference-bandwidth 10000
       network 192.168.0.0 0.0.0.7 area 0
       network 203.0.113.0 0.0.0.255 area 0
       default-information originate
      exit

Switch0

      vlan 10
       name VLAN10
      exit
      vlan 20
       name VLAN20
      exit
      interface range F0/1 - 4
       switchport mode access
       switchport access vlan 10
       no shutdown
      exit
      interface F0/5
       switchport mode trunk
       switchport trunk allowed vlan 10,20
       no shutdown
      exit

Switch1

      vlan 10
       name VLAN10
      exit
      vlan 20
       name VLAN20
      exit
      interface range F0/1 - 4
       switchport mode access
       switchport access vlan 20
       no shutdown
      exit
      interface F0/5
       switchport mode trunk
       switchport trunk allowed vlan 10,20
       no shutdown
      exit

PC0

      interface Fa0
       ip address 192.168.10.10 255.255.255.0
       default-gateway 192.168.10.1
      exit

PC1

      interface Fa0
       ip address 192.168.10.11 255.255.255.0
       default-gateway 192.168.10.1
      exit

PC2

      interface Fa0
       ip address 192.168.20.10 255.255.255.0
       default-gateway 192.168.20.1
      exit

PC3

      interface Fa0
       ip address 192.168.20.11 255.255.255.0
       default-gateway 192.168.20.1
      exit

Verification

The HSRP status on Router0 and Router1 shows Router0 as active for VLAN 10 (virtual IP 192.168.10.1) and Router1 as active for VLAN 20 (virtual IP 192.168.20.1). Failover was tested by shutting down Router0’s G0/0, confirming Router1 took over as active for VLAN 10.

Connectivity Test

A successful ping from PC0 (192.168.10.10) to PC2 (192.168.20.10) verifies inter-VLAN routing via OSPF. A ping from PC0 to Router2’s Loopback0 (203.0.113.1) confirms external connectivity through the default route. HSRP failover ensures uninterrupted connectivity during router failures.
