STP Toolkit Lab

Overview

This lab demonstrates the configuration and testing of Spanning Tree Protocol (STP) and its toolkit features (PortFast, BPDU Guard, Root Guard) on a network with three Layer 2 switches (Switch0, Switch1, Switch2) and a router (Router0) for inter-VLAN routing. The primary goal is to understand and implement STP to prevent network loops in a redundant Layer 2 topology while optimizing performance and enhancing security using Cisco’s STP toolkit.

What is STP? Spanning Tree Protocol, defined by the IEEE 802.1D standard, is a Layer 2 protocol that ensures a loop-free topology in Ethernet networks with redundant links. Without STP, broadcast storms and MAC table instability could cripple a network. STP achieves this by electing a root bridge, calculating the shortest path to the root for each switch, and blocking redundant ports to eliminate loops. The protocol operates by exchanging Bridge Protocol Data Units (BPDUs) to determine the network topology and adjust port roles (root port, designated port, or blocked port) dynamically.

STP Toolkit Features: Cisco enhances STP with additional features to address specific challenges in modern networks:

  • PortFast: Accelerates the transition of access ports (connected to end devices like PCs) to the forwarding state, bypassing the standard 30-50 second listening and learning phases. This reduces downtime for end devices but should only be enabled on ports that connect to a single host to avoid loops.
  • BPDU Guard: Protects PortFast-enabled ports by disabling them (putting them into an err-disabled state) if a BPDU is received, preventing rogue switches or misconfigurations from introducing loops. It’s a critical security feature for access-layer ports.
  • Root Guard: Prevents an unauthorized switch from becoming the root bridge by blocking a port if it receives a superior BPDU (indicating a lower bridge priority). This ensures the intended root bridge (e.g., Switch0 in this lab) remains in control, maintaining the designed topology.

This lab explores these features in a practical setting, demonstrating how they work together to create a stable, secure, and efficient network. By configuring and testing these tools, the lab provides hands-on experience with STP optimization and troubleshooting, key skills for network administration and CCNA certification preparation.

Topology

  • Switches: Switch0 (root bridge, priority 4096), Switch1 (priority 32778), Switch2 (priority 32778).
  • Connections:
    • Switch0 F0/1 to Switch1 F0/1 (trunk).
    • Switch0 F0/2 to Switch2 F0/2 (trunk).
    • Switch1 F0/3 to Switch2 F0/3 (trunk, blocked).
    • Switch0 F0/3 to Router0 G0/0/0 (trunk).
  • Access Ports:
    • Switch0 F0/4: PC0 (VLAN 10), Switch0 F0/5: PC1 (VLAN 20).
    • Switch1 F0/4: PC2 (VLAN 10), Switch1 F0/5: PC3 (VLAN 20).
    • Switch2 F0/4: PC4 (VLAN 10), Switch2 F0/5: PC5 (VLAN 20).
  • VLANs: VLAN 10 (IT Dept), VLAN 20 (Sales).
  • IP Addressing:
    • VLAN 10: 192.168.10.0/26, Gateway 192.168.10.62.
    • VLAN 20: 192.168.10.64/26, Gateway 192.168.10.126.

Configurations

Switch0 (Root Bridge)

configure terminal
spanning-tree vlan 10,20 priority 4096
interface FastEthernet0/1
 switchport mode trunk
 switchport trunk allowed vlan 10,20
exit
interface FastEthernet0/2
 switchport mode trunk
 switchport trunk allowed vlan 10,20
exit
interface FastEthernet0/3
 switchport mode trunk
 switchport trunk allowed vlan 10,20
exit
interface FastEthernet0/4

STP Analysis

Switch0 is the root bridge with priority 4096 (Root ID 4106 for VLAN 10). Switch2 F0/3 (to Switch1 F0/3) is blocked to prevent a loop in the triangle topology. Switch1 F0/1 (to Switch0) and Switch2 F0/2 (to Switch0) are root ports.

Toolkit Feature Tests

BPDU Guard Test

A rogue switch was connected to Switch0 F0/4, triggering BPDU Guard to put the port in an err-disabled state.

Root Guard Test

Switch2's priority was set to 0, causing Switch1 F0/3 to enter a root-inconsistent state due to Root Guard, protecting Switch0 as the root bridge.

Inter-VLAN Routing

A successful ping between PC0 (VLAN 10, 192.168.10.1) and PC3 (VLAN 20, 192.168.10.66) confirms inter-VLAN routing via Router0 using Router-on-a-Stick.
